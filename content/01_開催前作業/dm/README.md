# DM・Doorkeeper配信メッセージ運用

Doorkeeperには**ステップメール（登録日起点の自動連続配信）も名前差し込みもない**。
そこで「テンプレを用意 → トリガーで**手動一斉送信**」で、体験参加者との関係を保ち、
**「体験＝終わり」ではなく「続きのある一歩目」だと繰り返し伝える**ための運用をまとめる。

## なぜやるか（背景）
- 2026年7月初旬から、**体験 → 継続の引き上げ（成約B）**が低下。
- 参加者から「体験の分で終わりと思っていた」という声があった。
- Doorkeeperを入口に絞った結果、教材の幅・作品・継続の存在が伝わっていなかった。
- → 各接点で「続けられること」と「教材・作品」を**折々に見せて**誤解を解く。

## ファイルの種類

| 種別 | ファイル名 | 宛先 | タイミング |
|---|---|---|---|
| **募集DM** | `dm-YYYY-MM-DD.md` | メンバー登録済み・未参加など | 開催告知時 |
| **事前連絡** | `dm-YYYY-MM-DD-事前連絡.md` | その回の参加者 | 開催前（Doorkeeperリマインダーに追記）。**その回の特徴・進め方**で安心を作る（継続の種まきはしない） |
| **開催後フォロー** | `dm-YYYY-MM-DD-開催後フォロー.md` | その回の参加者 | 参加翌日〜2日後（日付指定で予約送信）。**体験＝入口**の本命 |
| **グループ一斉** | （定型化なし） | グループ全員 | ネタがある時（月1〜2目安） |

テンプレ: `dm-template-事前連絡.md` / `dm-template-開催後フォロー.md`
グループ一斉は、ネタが出たら都度、開催後フォローの文面を流用して送る。

## 毎回の共通ルール
- **タイトル（件名）を毎回書く**: ファイル先頭メタに `- タイトル:` を必ず入れる。Doorkeeper送信時にそのまま使えるようにする。
  - 募集DM: `【ブロマス】M/D（曜）…開催のご案内`
  - 事前連絡: `【ブロマス】M/D（曜）開催前日のご連絡`（リマインダー追記のみなら件名不要）
  - 開催後フォロー: `【ブロマス】ご参加ありがとうございました（M/D エリア）`
- **事前連絡**: 主催者の名乗り＋今回の特徴。継続・教材カタログは書かない（詳細は `dm-template-事前連絡.md`）。
- **開催後フォロー**: HPリンクは基本1通1つ（詰め込まない）。ローテーション候補:
  - カリキュラム一覧: `https://hoshihirotaka.github.io/PlayMakerLabLP/?utm_source=doorkeeper&utm_medium=followup#curriculum`
  - 作品（Instagram）: `https://www.instagram.com/block.master.game.lab/`
- **utm** を付ける（リンクを使うとき）: 事前連絡=`utm_medium=reminder` / 開催後フォロー=`utm_medium=followup`
- 継続プランの導線ページは現状「準備中／直接URL運用」のため、**リンクは張らず**柔らかく書く。開催後メールでは「当日お声がけ」ではなく「ご連絡ください」系にする。
- トーンは控えめ・簡潔。褒めすぎない。

## 使い方
1. 送る回の `dm-YYYY-MM-DD-事前連絡.md` / `dm-YYYY-MM-DD-開催後フォロー.md` を開く（なければテンプレから作る）。
2. 【 】部分（日付・教材名・会場・リンク）を差し替える。
3. 事前連絡＝Doorkeeperの自動リマインダー本文に追記／開催後フォロー＝日付指定で予約送信。
