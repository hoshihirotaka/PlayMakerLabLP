# Doorkeeper配信メッセージ運用

Doorkeeperには**ステップメール（登録日起点の自動連続配信）も名前差し込みもない**。
そこで「テンプレを用意 → トリガーで**手動一斉送信**」で、体験参加者との関係を保ち、
**「体験＝終わり」ではなく「続きのある一歩目」だと繰り返し伝える**ための運用をまとめる。

## なぜやるか（背景）
- 2026年7月初旬から、**体験 → 継続の引き上げ（成約B）**が低下。
- 参加者から「体験の分で終わりと思っていた」という声があった。
- Doorkeeperを入口に絞った結果、教材の幅・作品・継続の存在が伝わっていなかった。
- → 各接点で「続けられること」と「教材・作品」を**折々に見せて**誤解を解く。

## 3つの配信面（すべて総称宛・名前差し込みなし）

| 面 | 宛先 | タイミング（トリガー） | 役割 |
|---|---|---|---|
| **前日メール** | その回の参加者 | 開催前（Doorkeeperの自動リマインダーに追記） | 流れ・持ち物＋「続けられる」種まき＋作品リンク |
| **後日フォロー** | その回の参加者 | 参加翌日〜2日後（日付指定で予約送信） | **本命**。体験＝入口と明言し、教材の幅・継続の道を提示 |
| **グループ一斉** | グループ全員（未参加フォロワー含む） | ネタがある時（月1〜2目安） | 最近の作品・教材を1つ紹介し、離れた人にも見せ続ける |

※前日メール・後日フォローのテンプレは本フォルダに用意（`_template-前日メール.md` / `_template-後日フォロー.md`）。
　各回版は `YYYY-MM-DD.md`（前日・後日をまとめて記載）。
※グループ一斉は定型化せず、ネタ（新作品・新教材）が出たら都度、後日フォローの文面を流用して送る。

## 毎回の共通ルール
- **HPリンクを1通につき1つだけ**差し込む（詰め込まない）。ローテーション候補:
  - カリキュラム一覧: `https://hoshihirotaka.github.io/PlayMakerLabLP/?utm_source=doorkeeper&utm_medium=followup#curriculum`
  - 作品（Instagram）: `https://www.instagram.com/block.master.game.lab/`
- **utm** を付ける: 前日=`utm_medium=reminder` / 後日=`utm_medium=followup`（後でアナリティクスを入れたとき効果が見える）。
- 継続プランの導線ページ（オンライン・自宅・無料相談）は現状「準備中／直接URL運用」のため、**リンクは張らず**「当日お声がけ／次回案内」と柔らかく書く。
- トーンは控えめ・簡潔（既存DM `content/01_開催告知/dm/` を踏襲）。褒めすぎない。

## 使い方
1. 送る回の `YYYY-MM-DD.md` を開く（なければテンプレから作る）。
2. 【 】部分（日付・教材名・会場・リンク）を差し替える。
3. 前日メール＝Doorkeeperの自動リマインダー本文に追記／後日フォロー＝日付指定で予約送信。
