# 教材リポジトリ用プロンプト（カリキュラムデータ生成）

> このファイルは「教材リポジトリ側のエージェント」に貼り付けて使うための指示書です。
> LP（ブロマスゲームラボ公式サイト）のカリキュラム一覧セクションに表示するデータを、
> 構造化JSONとして出力してもらうことが目的です。
> LP側はこのJSONを受け取って描画するだけにするため、**出力フォーマットを厳密に守ってもらう**必要があります。
> 下の「--- ここから貼り付け ---」以降をコピーして渡してください。

## 双方向フィードバックの仕組み（運用者向け・貼り付け不要）

LPリポジトリと教材リポジトリが、それぞれ「相手への申し送りMarkdown」を1枚ずつ持ち、往復させて両方を改善する。

- **LP → 教材**: `curriculum/curriculum-feedback.md`（このリポジトリ。LP側が書く）
  … 受領ログ・表示側の要望・gaps優先度・教材側からの質問への回答
- **教材 → LP**: `curriculum-for-lp.md`（教材リポジトリ側。教材エージェントが書く）
  … 今回の変更点・除外/要判断・gaps・LP側への質問

### 往復サイクル
1. **LP→教材**: LP側が `curriculum-feedback.md` を更新 → 運用者が「このプロンプト＋feedbackの中身」を教材エージェントに渡す
2. **教材→LP**: 教材エージェントが `curriculum.json` を再生成 ＋ `curriculum-for-lp.md` を更新 → 運用者が両方をLPへ
3. **LP取り込み**: LP側が `curriculum.json` を取り込み（`curriculum.js` を再生成）＋ 相手の申し送りを読んで `curriculum-feedback.md` を更新 → 1へ戻る

## --- ここから貼り付け ---

あなたはこの教材リポジトリの内容を最もよく理解しているエージェントです。
この教材群を、一般公開されているLP（子ども向けゲームクリエイター教室「ブロマスゲームラボ」の公式サイト）の
「カリキュラム一覧」セクションに掲載するため、下記の仕様に従って**構造化データ（JSON）**を出力してください。

### ワークフロー（毎回の運用・重要）
このやり取りは、LPリポジトリと教材リポジトリの**双方向フィードバック**の一部です。以下を必ず行ってください。

1. **LP側からの申し送りを読む**: この指示の後に「LPからの申し送り（`curriculum-feedback.md` の中身）」が貼られている場合があります。そこに書かれた要望・質問・gaps優先度に**必ず目を通し**、可能な範囲で `curriculum.json` に反映してください。
2. **教材側からLPへの申し送りを書く**: 教材リポジトリ内に **`curriculum-for-lp.md`** というMarkdownを作成／更新し、以下を記載してください。
   - 今回の `curriculum.json` の変更点（前回から追加/更新/削除した教材）
   - 除外した教材・分類に迷った教材（＝ `meta.note` と同内容でよい）
   - `gaps`（足りない教材）と、その理由
   - **LP側への質問・要望**（例: この教材の表示名はこれでよいか、アイコン希望、優先して作るべきものの相談など）
   - LP側からの前回の要望への**対応可否・回答**
3. これにより、教材側は教材の充足を、LP側は表示・集客観点の改善を、相互に渡して両方を育てます。

### 背景・読み手
- 読み手は**小学生の保護者**です。専門用語は避け、何が身につくか・何を作るかが一読で伝わる説明にしてください。
- LPは静的サイト（ビルドなし）で、このJSONをそのまま読み込んでカードを描画します。
- 表示は「科目タブ（Roblox / AI）＋ 各教材カードの詳細アコーディオン」を想定しています。

### 用語のルール（重要）
- **コース**＝科目の区分（Roblox・AI）。データ上は `subject`。
- **プラン**＝通い方の区分（通常プラン・個別プラン）。**教材の説明文で「プラン」という語は使わないでください**（混同を避けるため）。
- 「初級/中級」という表記は使いません。難易度ではなく、下記の**タグ**で性質を表します。

### タグの定義（`tags`）
各教材に、以下から**1つ以上**（複数可）を付与してください。
- `体験用`: 初めての方向け。基本操作が中心で、**プログラミングなし**で「作る楽しさ」を体験する教材。
- `プログラミング`: スクリプトやロジックを使って、ギミック・仕組み・動きを作る教材。
- `クリエイティブ`: デザイン・世界観・見た目・演出など「表現」に重きを置く教材。

（例: アスレチック教材が、基本操作でも遊べて、かつ発展でギミックも学べるなら `["体験用", "プログラミング"]` のように複数付与）

### 出力フォーマット（厳守）
`curriculum.json` という名前で、以下のスキーマの**有効なJSONのみ**を出力してください。コメントや説明文をJSON内に混ぜないこと。

```json
{
  "meta": {
    "generatedAt": "2026-07-13",
    "sourceRepo": "＜この教材リポジトリ名＞",
    "note": "補足があれば自由記述（なければ空文字）"
  },
  "materials": [
    {
      "id": "roblox-athletic",
      "name": "アスレチック",
      "subject": "Roblox",
      "tags": ["体験用", "プログラミング"],
      "icon": "🏔️",
      "sessions": 1,
      "summary": "地形と仕掛けでゴールを目指す王道。",
      "detail": "ブロックや仕掛けを配置して、最後まで遊べるアスレチックゲームを完成させます。慣れてきたら動く床や回転ギミックにも挑戦できます。",
      "status": "published"
    }
  ],
  "gaps": [
    {
      "suggestedName": "＜あると良い教材名＞",
      "subject": "AI",
      "tags": ["クリエイティブ"],
      "reason": "なぜ必要か・どんな層/進度の穴を埋めるかを簡潔に"
    }
  ]
}
```

### 各フィールドの規則
- `id`: 一意な半角英小文字＋ハイフン（kebab-case）。`科目-内容` の形が望ましい（例 `roblox-treasure`, `ai-sidescroller`）。
- `name`: 教材名（日本語・短く）。
- `subject`: `"Roblox"` または `"AI"` のいずれか。該当しない科目があれば `note` で報告。
- `tags`: 上記3種からの配列。最低1つ。許可された値以外を入れない。
- `icon`: 内容を表す絵文字**1つ**（任意。なければ空文字 `""`）。
- `sessions`: 完了までのコマ数。数値（例 `1`）。幅がある場合は文字列（例 `"1〜2"`）。不明なら `"未定"`。
- `summary`: カード表面用。**40字以内目安**、1文、保護者向け。
- `detail`: アコーディオン展開用。**120字以内目安**。作るもの・身につくことを具体的に。任意（なければ空文字）。
- `status`: 公開可能なら `"published"`、準備中なら `"coming_soon"`。

### `gaps`（足りない教材）について
- 現状の教材群を俯瞰し、「あると良いが今はない教材」を `gaps` に列挙してください。
- 判断軸の例: 体験用→プログラミング→クリエイティブの進度がつながっているか / 各科目でタグの偏りがないか / 人気ジャンルの抜け。
- 各項目に**不足の理由**（`reason`）を必ず添えてください。ないと判断したら空配列 `[]` で構いません。

### 守ってほしいこと
- **実在する教材のみ**を `materials` に入れること。存在しない教材を創作しない。判断に迷うものは `note` で申し送る。
- 全教材を網羅すること（一部だけにしない）。数が多くても全件出す。
- 最終出力は「`curriculum.json` の中身（純粋なJSON）」＋「`curriculum-for-lp.md` の更新」＋「人間向けの短いサマリー（何件・科目別内訳・gaps件数）」。JSON本体にサマリーを混ぜないこと。
