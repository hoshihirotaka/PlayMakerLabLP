## コンテンツ制作・マーケティングの指針

このセクションは、LP、バナー、チラシ、SNS投稿を作成する際の「魂」となるガイドラインです。

### 1. 核心となる思想 (Core Philosophy)
- **「プログラミング」ではなく「ゲームの作り方」を教える**: 
  文法ではなく「どうすれば面白くなるか」という企画・設計（プランニング）を主役に据える。
- **ショートカットの肯定**: 
  大人が20歳から始めた苦労を、今の子供はAIを使って10分で飛び越える。その「時間の解放」を価値とする。
- **目的主導型の学習**: 
  「このギミックを作りたいから学ぶ」という、遊びから逆算した学びを推奨する。

### 2. ターゲットペルソナ（解像度）
- **子供**: 
  小3でAIと壁打ちを始め、勝手に「企画会議」を始めてしまうような、好奇心の塊。Robloxという日常（遊び場）を自分の手で作り変えたいと願っている層。
- **保護者**: 
  35-44歳の共働き世帯、または教育熱心な層。子供に「消費」だけでなく「創造」の力を身につけさせたいと願っている。

### 3. 反応率を高めるフレームワーク (DECA)
制作物には以下の要素を同時多発的に含めること：
- **Discovery (発見)**: 「小3がAIとゲーム企画を始めた実話」など、常識を覆すエピソード。
- **Engage (関与)**: 「君がいつも遊んでいるRobloxを、君専用に改造しよう」。
- **Check (確認)**: 「Windows 5台・Mac 3台のプロ仕様環境」「現役プロ開発者による直接指導」。
- **Action (行動)**: 「まずは近所のワークショップへ（保育園から徒歩5分、地域密着）」。

### 4. 文章トーン＆マナー
- **熱量を可視化する**: 
  整いすぎた「塾のチラシ」ではなく、開発現場のワクワク感が伝わるライブ感のある言葉を使う。
- **比較対象**: 
  「ただのゲーム遊び」vs「世界へ届けるクリエイター」。10代でヒットを出す海外事例を引き合いに出し、日本からの挑戦を促す。

## イベント管理の仕様

### 仕組み

`index.html` にイベントをハードコードせず、イベントごとに独立したJSファイルで管理する。
ファイルを追加するだけでLPに反映される。

```
events/
  2026-06-26.js   ← イベントごとに1ファイル
  2026-XX-XX.js   ← 次回以降はここに追加
```

### イベント追加手順

**① `events/YYYY-MM-DD.js` を作成する**（既存ファイルをコピーして編集）

```js
(window.EVENTS = window.EVENTS || []).push({
  id: "2026-08-XX",           // ファイル名と合わせる（ソート順に使用）
  date: "2026年8月XX日（X）",
  datetime: "2026年8月XX日(X) HH:MM-HH:MM",
  location: "会場名",
  area: "エリア名",
  address: "〒XXX-XXXX 住所",
  target: "小学1年生〜中学3年生（保護者参加OK）",
  capacity: "Roblox X名 / AI X名（予約優先）",
  equipment: "パソコン持参大歓迎（会場でも用意あり）",
  doorkeeperUrl: "https://gameschool.doorkeeper.jp/events/XXXXXX",
  timetable: [
    { time: "HH:MM-HH:MM", label: "Roblox" },
    { time: "HH:MM-HH:MM", label: "AI" }
  ]
});
```

**② `index.html` の所定箇所に1行追加する**

```html
<!-- ▼ イベント追加時はここに1行追加する（id昇順で並びます） -->
<script src="events/2026-06-26.js"></script>
<script src="events/2026-XX-XX.js"></script>  ← 追加
```

**③ `index.html` の `<head>` 内 JSON-LD を更新する**（構造化データ・SEO用）

`@id` と各日時・URLをイベントに合わせて追記する。

### 注意事項

- イベントは `id`（YYYY-MM-DD）の昇順で自動ソートされ、初期表示は「今日以降で最も近い開催日」が `open` 状態になる（すべて過去日の場合は最新日を `open`）
- `index.html` の変更は `<script>` 1行追加のみのため、ブランチ間のコンフリクトはほぼ発生しない
- 各イベントファイルは独立しているため、編集しても他のイベントに影響しない
- ローカル確認はブラウザで `index.html` を直接開けばよい（サーバー不要）
