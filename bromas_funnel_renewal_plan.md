# ブロマス 集客導線 統合改修プラン

> 対象：LP（PlayMakerLabLP）／Doorkeeperイベント説明文（v2ドラフト）／Doorkeeperコミュニティページ
> 背景：進藤さんFB「教材が複数あることが伝わらない」＋ CoderDojoのutm調査結果（2026/6/12）

---

## 0. 全体方針：3つの場所の役割分担

| 場所 | 役割 | 読者の状態 |
|---|---|---|
| **Doorkeeperイベントページ** | 今回の開催情報・申込（ランディング） | 申し込むか決めようとしている |
| **LP（ホームページ）** | カリキュラム図鑑・信頼・実績（バックエンド） | もっと詳しく知りたい・主催者を確かめたい |
| **Doorkeeperコミュニティ** | メンバー登録・継続接点 | 今回は行けないが興味あり |

**核心の整理**：教材ラインナップの常設展示は**LPの仕事**にする。
Doorkeeperイベント文は「今回の教材」＋「毎回変わります、一覧はこちら→LP」で短く保つ。
これでv2ドラフトの「まとめ型 vs 切り替え型」問題は**両取り**で解決し、
Doorkeeper→LPの導線（信頼チェック動線）も自然に生まれる。

## 1. LP改修（PlayMakerLabLP）

### 1-1. カリキュラム一覧セクションの新設【最優先・進藤さんFB対応】
- 現状の問題：Robloxコース＝「アクション制作」固定表記／完成済み教材群が「追加準備中」と未来形
- カード形式で常設展示。子供に通じる言葉で：

| 表示名（案） | 教材 | 一言説明 |
|---|---|---|
| アスレチック | obby系 | 地形と仕掛けでゴールを目指す王道 |
| たからさがし | Find系 | マップとアイテム配置で「探す楽しさ」を設計 |
| レース | レース系 | スピード感とコース設計で競争を作る |
| ミニゲームパーティ | Party系 | みんなで遊べる小さいゲームを詰め込む |
| アバターショップ | アバター/UGC | 着せ替えアイテムとお店を作る |

- 各カードに「体験会で選べる／継続カリキュラムあり」のバッジ
- セクション末尾に「**回によってテーマが変わります。2回目以降も新しい教材で参加できます**」を明記
- ※表示名・ラインナップはひったさん確定。準備中のものは「COMING SOON」カードで正直に区別

### 1-2. FAQ拡充（現状3問→7問程度)
追加候補：
- 「作りたいものが思いつかなくても参加できますか？」→ 決められたテーマに沿って作ることもできます（朝メモの文言。DMにも転用）
- 「2回目の参加でも楽しめますか？」→ v2ドラフトの回答を流用
- 「継続的な教室はありますか？」→ v2ドラフトの回答を流用（通常プラン/個別プラン）
- 「料金はいくらですか？」→ 体験会（Roblox 1,900円/AI 900円）とプランの違い

### 1-3. その他
- 固定ヘッダーナビ追加（カリキュラム／作品ギャラリー／参加者の声／日程／講師／FAQ）
- 「3人に1人がリピート」をヒーロー付近で強調（継続の証拠）
- ビフォーアフター：リピーターの子1組の「1回目→3回目」を作品ギャラリーに追加（Canva作品集から流用）
- ヒーローの「埼玉・南与野 開催」表記が現会場と合っているか確認
- リンク総点検：リポジトリで `tickets/new` と utm をgrep。イベントJSONのDoorkeeperリンクを
  イベントページ直＋ `utm_source=lp&utm_medium=referral&utm_campaign=bromas` に統一。
  フッターのDoorkeeperコミュニティリンクにも utm（`utm_medium=footer`）

## 2. Doorkeeperイベント説明文（v2ドラフトへの修正提案）

### 2-1. 構成の変更（役割分担に合わせる）
- 「🎯 今回のRobloxの教材」（1行差し替え）：**残す**
- 「📚 初めてのお子様向け教材は毎回変わります」のラインナップ列挙：**3行→1行に圧縮し、LPへリンク**
  > 教材は複数あり、回ごとに切り替えています。ラインナップは[公式サイトのカリキュラム一覧](LPのURL?utm_source=doorkeeper&utm_medium=event)をご覧ください。
- これで切り替え型の短さと、まとめ型の安心感を両立。差し替えポイントも【教材1行】と【スケジュール表】の2箇所のまま

### 2-2. 追記・修正
- 主催者紹介の末尾にLPへのリンク追加：
  > 教室の詳しい紹介・作品ギャラリーは[公式サイト](LPのURL?utm_source=doorkeeper&utm_medium=event)へ
- FAQは良い出来なのでそのまま。LP側FAQ（1-2）と回答文面を共通化（コピペ運用でズレ防止）

### 2-3. 申込フォームに設問追加【計測の本命】
- Doorkeeperの申込フォームカスタム設問：
  > このイベントをどこで知りましたか？
  > （インスタ広告／インスタの投稿・プロフィール／公式サイト／紹介・口コミ／チラシ／その他）
- ブロマスDoorkeeperにはGAがないため、帰属計測はこれが主役

## 3. Doorkeeperコミュニティページ

- 自己紹介の「子供向けのプログラミング教室も運営」→「**子供向けのゲームクリエイター教室を運営**」に修正（言葉の定義表に準拠）
- LPへのリンクを追加（「ブロマスゲームラボについて」の末尾）：
  > 教室の詳しい紹介・カリキュラム・作品ギャラリーは[公式サイト](LPのURL?utm_source=doorkeeper&utm_medium=community)へ
- 「直近の風景」写真の更新（3月のまま→直近回のものに。更新頻度は四半期に1回でOKとルール化）
- それ以外は現状維持（理念・展望は良い状態）

## 4. 計測まわり（utm・テスト）

### utm付け分け（ブロマス版）
| 発行場所 | utm |
|---|---|
| LP→Doorkeeper（イベントJSON） | `utm_source=lp&utm_medium=referral&utm_campaign=bromas` |
| LPフッター→コミュニティ | `utm_source=lp&utm_medium=footer` |
| Doorkeeperイベント文→LP | `utm_source=doorkeeper&utm_medium=event` |
| コミュニティページ→LP | `utm_source=doorkeeper&utm_medium=community` |
| インスタプロフィール→LP | `utm_source=instagram&utm_medium=profile` |
| DM→Doorkeeper | `utm_source=instagram&utm_medium=dm` |
| 広告 | ブースト自動付与に任せる（ig/paid） |

※Doorkeeper側はGAがないため、Doorkeeper行きutmは「将来GAを入れた時用の仕込み」。
　現時点の帰属はフォーム設問（2-3）で取る。LP行きutmはLPのGAで即計測可能。

### 広告テスト
- 次回出稿から**期間分けテスト**：今回は直行のみ→次回はLP経由のみ
- 比較指標：申込数÷消化金額（CPA）＋フォーム設問の回答
- 並走A/BはGAがないため行わない

## 5. 表記整合ルール

- 教材表示名（1-1の表）は **LP・Doorkeeper・DM・インスタで同一表記**を使う

- FAQの回答文はLPとDoorkeeperで共通文面（片方を直したら必ずもう片方も）
